# 📖 MYMeal AI 使用指南

## 🚀 三步快速开始

### 第1步：配置 Supabase

1. 访问 [supabase.com](https://supabase.com) 创建项目
2. 复制 `.env.local.example` 为 `.env.local`
3. 填入你的 Supabase URL 和 API Keys（记得用 Legacy JWT 密钥！）
4. 在 SQL Editor 中执行：
   - `supabase/schema.sql`
   - `supabase/seed-recipes.sql`
5. 启用 Authentication → Anonymous Sign-ins

### 第2步：配置豆包 API

在 `.env.local` 中添加：
```env
DOUBAO_API_KEY=你的密钥
DOUBAO_MODEL=你的模型
```

### 第3步：启动应用

**Windows**：双击 `start.bat`  
**macOS**：双击 `start.command`  
**Linux**：运行 `./start.sh`

或手动运行：
```bash
npm install
npm run dev
```

## 📋 主要功能

### ✅ 已实现
- 个人档案设置（年龄、性别、健康目标、预算等）
- 🤖 **豆包AI智能生成** 7 天膳食计划（每次都不同！）
- 90+ 道马来西亚特色食谱
- 🛒 自动生成购物清单（AI 驱动，后台异步）
- 替换菜肴功能（智能推荐同类型菜品）
- 营养分析仪表盘
- 登录/注册功能
- 匿名用户支持
- 加载进度显示

### 🎯 使用流程
1. 首页填写个人信息
2. 点击"生成膳食计划"
3. 等待 AI 生成（约 30-60 秒）
4. 查看 7 天膳食计划
5. 点击菜肴卡片查看详情
6. 点击"替换菜肴"更换不喜欢的菜品
7. 访问"购物清单"查看所需食材
8. 访问"营养分析"查看健康数据

## 🔧 故障排除

| 问题 | 解决方案 |
|------|----------|
| 点击生成按钮没反应 | 检查 Supabase 配置和匿名登录 |
| 购物清单为空 | 检查豆包 API 配置 |
| 加载很慢 | 正常现象，AI 生成需要时间 |
| 404 错误 | 刷新页面或清除浏览器缓存 |
| 启动脚本不运行 | macOS 用户用 .command，Windows 用 .bat |

## 💡 小贴士

1. **第一次生成会比较慢**：系统需要计算营养、匹配食谱、生成购物清单
2. **多次点击按钮无效**：请等待当前操作完成
3. **替换菜肴**：只会替换为同类型（早/午/晚餐）且未在本周计划中的菜品
4. **购物清单**：由 AI 智能分析菜谱并生成，包含数量和备注
5. **深色模式**：系统会自动适配系统主题
