# ✅ 部署检查清单

在部署前，请确认以下项目：

## 📦 代码准备

- [ ] 代码已推送到 GitHub
- [ ] `.env.local` 文件**没有**被提交到 Git（已在 .gitignore 中）
- [ ] 所有依赖已安装（`npm install` 成功）

## 🔐 Supabase 配置

- [ ] Supabase 项目已创建
- [ ] 已执行 `schema.sql` 创建所有表
- [ ] 已执行 `seed-recipes.sql` 导入食谱数据
- [ ] 已添加 `notes` 列到 `shopping_list_items` 表
- [ ] 已启用匿名登录（Authentication → Providers → Anonymous Sign-ins）
- [ ] 已获取 Project URL 和 anon key

## 🌐 环境变量

在 Vercel/Netlify 中需要设置以下环境变量：

```
NEXT_PUBLIC_SUPABASE_URL=https://你的项目.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=你的anon密钥
```

> ⚠️ **重要**：确保在 Production、Preview 和 Development 三个环境都添加了这些变量

## 🧪 本地测试

部署前，先在本地测试：

```bash
# 1. 确保有 .env.local 文件
# 2. 构建项目
npm run build

# 3. 启动生产模式
npm start

# 4. 访问 http://localhost:3002 测试
```

如果本地测试通过，部署到线上也应该没问题。

## 🚀 部署步骤

1. [ ] 在 Vercel/Netlify 创建项目
2. [ ] 连接 GitHub 仓库
3. [ ] 添加环境变量
4. [ ] 开始部署
5. [ ] 等待构建完成
6. [ ] 测试网站功能

## ✅ 部署后测试

- [ ] 首页能正常加载
- [ ] 可以填写个人信息表单
- [ ] 可以生成膳食计划
- [ ] 可以查看购物清单
- [ ] 所有页面样式正常显示
- [ ] 没有控制台错误

## 🔍 如果遇到问题

1. **检查部署日志**：查看构建过程中的错误信息
2. **检查环境变量**：确认所有变量都已正确设置
3. **检查浏览器控制台**：按 F12 查看是否有 JavaScript 错误
4. **检查 Supabase**：确认数据库连接正常

---

**完成所有检查项后，就可以开始部署了！** 🎉
